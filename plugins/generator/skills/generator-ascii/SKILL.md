# บทบาท:
คุณทำหน้าที่เป็นเครื่องมือสร้าง ASCII art ให้ผู้ใช้ ครอบคลุม 4 โหมด: แปลงข้อความเป็น
banner ตัวใหญ่ (สไตล์ figlet/toilet), แปลงรูปภาพเป็นภาพตัวอักษร (สไตล์
jp2a/chafa/img2txt), พูดข้อความผ่านตัวการ์ตูน (สไตล์ cowsay) และเติมสีรุ้งไล่เฉด
(สไตล์ lolcat)

**หัวใจของ skill นี้คือทำงานแบบ self-contained** — แทนที่จะเรียกโปรแกรม `figlet`,
`toilet`, `jp2a`, `chafa`, `img2txt`, `cowsay`, `lolcat` ที่ต้องติดตั้งแยกต่างหาก
(และพฤติกรรม/ฟอนต์อาจไม่เหมือนกันในแต่ละเครื่อง) skill นี้ bundle script ของตัวเอง
ไว้ใน `scripts/` ที่ทำหน้าที่เดียวกัน ใช้งานได้ทันทีทุกเครื่องที่มี Python

**ข้อควรรู้เรื่องฟอนต์:** ฟอนต์ตัวอักษรใหญ่ที่ใช้ใน `text_banner.py` เป็น bitmap
5x7 ที่ออกแบบขึ้นเองให้อ่านง่ายและได้ฟีลแบบ figlet ไม่ใช่การ copy ฟอนต์จริงของ
figlet มา — ถ้าผู้ใช้ต้องการฟอนต์ลาย/สไตล์เพิ่มเติม ให้บันทึกไว้เป็นข้อเสนอปรับปรุง
skill ในอนาคต แทนที่จะพยายามเลียนแบบฟอนต์อื่นแบบเดา

# รูปแบบ:

## ขั้นตอนที่ 1 — เลือกวิธีเรียก Python

เช็คว่าเครื่องมี `uv` หรือไม่ (`command -v uv` บน macOS/Linux, `Get-Command uv` บน
Windows PowerShell):
- **ถ้ามี `uv`** ให้ใช้ `uv run scripts/<ชื่อ script>.py ...` เสมอ — `uv` จะจัดการ
  dependency ให้อัตโนมัติผ่าน inline script metadata ที่แนบไว้ในไฟล์ (มีผลกับ
  `image_to_ascii.py` ที่ต้องใช้ Pillow เท่านั้น สคริปต์อื่นไม่มี dependency ภายนอก)
- **ถ้าไม่มี `uv`** ให้ fallback ไปเรียก `python3`/`python` ตรงๆ (macOS/Linux) หรือ
  `py -3` (Windows) — ใช้ได้ทันทีกับ `text_banner.py`, `cowsay.py`, `rainbow.py`
  เพราะไม่มี dependency ภายนอก ส่วน `image_to_ascii.py` ต้องมี Pillow ติดตั้งไว้ก่อน
  (`pip install pillow`) ถ้ายังไม่มีและผู้ใช้ไม่อยากติดตั้ง uv ให้แจ้งวิธีติดตั้งสั้นๆ

## ขั้นตอนที่ 2 — แยกประเภทคำขอ

- มีแต่ **ข้อความ** และอยากได้ตัวใหญ่/banner → โหมด Text Banner (`text_banner.py`)
- มี **ไฟล์รูปภาพ** แนบมาหรือระบุ path มา → โหมด Image to ASCII (`image_to_ascii.py`)
- ขอให้ "วัวพูด" หรือให้ตัวการ์ตูนพูดข้อความ → โหมด Cowsay (`cowsay.py`)
- ขอสีรุ้ง/เติมสีให้ ASCII art ที่มีอยู่ → โหมด Rainbow (`rainbow.py`) ใช้ประกอบกับ
  โหมดอื่นก็ได้ (pipe output ของ 3 โหมดข้างบนเข้า `rainbow.py` ต่อ)

ถ้าผู้ใช้ไม่ได้ระบุว่าต้องการโหมดไหนชัดเจน (เช่นพิมพ์มาแค่ "ทำ ASCII art ให้หน่อย")
ให้ถามสั้นๆ ว่าต้องการแปลงข้อความหรือรูปภาพ

## ขั้นตอนที่ 3 — Text Banner (สไตล์ figlet/toilet)

```
uv run scripts/text_banner.py "ข้อความ" [--char '#'] [--scale 1] [--spacing 1]
```

- `--char` เปลี่ยนตัวอักษรที่ใช้เติมลาย (default `#`) — ลองใช้ `█`, `@`, `$` ให้ฟีล
  ต่างกันได้ (สไตล์ `toilet` มักหนาและมีสี ให้เลือก `--char '█'` แล้วต่อด้วย rainbow
  ถ้าผู้ใช้อยากได้ฟีลแบบ toilet)
- `--scale` ขยายขนาดตัวอักษร (2 = ใหญ่ขึ้นเท่าตัว)
- รองรับ A-Z, 0-9, เว้นวรรค และเครื่องหมาย `! ? . , ' -  :` เท่านั้น — ตัวพิมพ์เล็ก
  จะถูกแปลงเป็นตัวพิมพ์ใหญ่อัตโนมัติ ตัวอักษรที่ไม่รองรับ (เช่นภาษาไทย, emoji) จะ
  render เป็นช่องว่าง — ถ้าผู้ใช้ขอข้อความภาษาไทย ให้แจ้งข้อจำกัดนี้ก่อนรัน

## ขั้นตอนที่ 4 — Image to ASCII (สไตล์ jp2a/chafa/img2txt)

```
uv run scripts/image_to_ascii.py <path/to/image> [--width 100] [--color]
```

- ไม่ใส่ `--color` → output ตัวอักษรธรรมดาโทนเดียว (ฟีลแบบ `jp2a`/`img2txt`)
- ใส่ `--color` → output พร้อมสี ANSI truecolor ตามสีจริงของรูป (ฟีลแบบ `chafa`) — สี
  จะแสดงถูกต้องเมื่อรันใน terminal จริง ผลลัพธ์ที่ capture มาแสดงในแชทอาจเห็นเป็น
  escape code ดิบถ้า UI ไม่ render ANSI ให้แจ้งผู้ใช้ว่าควรรันเองใน terminal เพื่อเห็นสี
- `--width` ควบคุมความกว้าง (จำนวนตัวอักษรต่อแถว) ยิ่งกว้างยิ่งละเอียดแต่ยาวขึ้นตาม
  ปรับตามความเหมาะสม (รูปสี่เหลี่ยมจัตุรัสมักดูดีที่ width ~80-120)
- รองรับไฟล์รูปภาพทุกฟอร์แมตที่ Pillow อ่านได้ (JPG, PNG, GIF, BMP, WebP ฯลฯ)

## ขั้นตอนที่ 5 — Cowsay (พูดผ่านตัวการ์ตูน)

```
uv run scripts/cowsay.py "ข้อความ" [--width 40] [--eyes 'oo'] [--think]
```

- `--eyes` เปลี่ยนลูกตา 2 ตัวอักษร เช่น `OO`, `--`, `**`, `$$` ให้เข้ากับอารมณ์ข้อความ
- `--think` เปลี่ยนจากลูกโป่งคำพูดเป็นลูกโป่งความคิด (วงกลม แทนหางแหลม)
- ข้อความยาวจะ word-wrap อัตโนมัติตาม `--width`

## ขั้นตอนที่ 6 — Rainbow (เติมสีรุ้ง สไตล์ lolcat)

```
uv run scripts/text_banner.py "TEXT" | uv run scripts/rainbow.py
uv run scripts/rainbow.py --file output.txt
uv run scripts/rainbow.py "ข้อความสั้นๆ"
```

ใช้ pipe ต่อจาก `text_banner.py` หรือ `cowsay.py` ได้ตรงๆ เพื่อให้ output มีสีรุ้ง
ไล่เฉด (เหมือนคำสั่ง `figlet ... | lolcat` จริง) `--freq` ปรับความเร็วการหมุนเฉดสี
ตามแนวตั้ง, `--spread` ปรับความถี่การไล่สีตามแนวนอน

## ขั้นตอนที่ 7 — แสดงผลและบันทึกไฟล์ (ถ้าต้องการ)

แสดงผลลัพธ์ในคำตอบให้ผู้ใช้เห็นเสมอ (ครอบด้วย code block ธรรมดา — โหมดที่ไม่มีสี
จะอ่านง่ายในแชท ส่วนโหมดมีสีแนะนำให้บอกผู้ใช้ว่าเห็นสีเต็มๆ ได้เมื่อรันเองใน terminal)
ถ้าผู้ใช้อยากได้ไฟล์ (เช่นเอาไปแปะใน README, comment, หรือ signature) ให้ redirect
output ไปเป็นไฟล์ `.txt` ให้ (`> banner.txt`) แทนการ copy ด้วยมือ

# คำขอ:
- ห้ามสมมติว่าเครื่องผู้ใช้มี `figlet`/`toilet`/`jp2a`/`chafa`/`img2txt`/`cowsay`/
  `lolcat` ติดตั้งอยู่ — ใช้ script ใน `scripts/` เสมอ ไม่ใช่เรียกโปรแกรมเหล่านี้ตรงๆ
- เช็คว่ามี `uv` ก่อนเสมอตามขั้นตอนที่ 1 แล้วเลือกวิธีเรียก python ให้เหมาะกับเครื่อง
- โหมด Image to ASCII ต้องมี path ไฟล์รูปภาพจริงเสมอ ถ้าผู้ใช้ยังไม่แนบมา ให้ถามก่อน
- ข้อความสำหรับ Text Banner ที่มีอักขระนอกเหนือ A-Z/0-9/เครื่องหมายพื้นฐาน (เช่น
  ภาษาไทย, emoji) จะ render ไม่ได้ครบ — แจ้งข้อจำกัดนี้ก่อนรันเสมอ อย่ารันเงียบๆ
  แล้วปล่อยให้ผู้ใช้เห็นช่องว่างเอง
- ไม่ต้องขออนุญาตก่อนรัน script เหล่านี้ (เป็น read-only ต่อ working directory เดิม
  ยกเว้นตอน redirect output เป็นไฟล์ใหม่ตามที่ผู้ใช้ขอ)

# ไฟล์แนบ:
- โหมด Image to ASCII: path ของไฟล์รูปภาพที่ต้องการแปลง (จำเป็น)
- โหมดอื่น: ข้อความที่ต้องการแปลง/ให้ตัวการ์ตูนพูด (จำเป็น)
