# ตัวอย่างผลลัพธ์ที่ถูกต้อง (Few-shot Examples)

ใช้ตัวอย่างเหล่านี้เป็น reference สำหรับ tone, ความยาว และสไตล์การเขียนที่ถูกต้อง

---

CVE (Common Vulnerabilities and Exposures) คือระบบรหัสมาตรฐานที่ใช้ระบุและอ้างอิงช่องโหว่ด้านความปลอดภัยในซอฟต์แวร์หรือระบบไอทีแต่ละรายการแบบไม่ซ้ำกัน ทำให้ผู้พัฒนา ผู้เชี่ยวชาญด้านความปลอดภัย และองค์กรต่าง ๆ สามารถสื่อสารและเข้าใจตรงกันเกี่ยวกับช่องโหว่เดียวกันได้ โดยระบบนี้ถูกดูแลโดย MITRE และมักอยู่ในรูปแบบรหัส เช่น CVE-ปี-ลำดับ ซึ่งเปรียบเสมือน "หมายเลขประจำตัว" ของช่องโหว่นั้น ๆ

---

SLA (Service Level Agreement) คือข้อตกลงระหว่างผู้ให้บริการกับลูกค้าที่กำหนดมาตรฐานการให้บริการแบบวัดผลได้ เช่น ความพร้อมใช้งานของระบบ เวลาตอบสนอง และเวลาแก้ไขปัญหา พร้อมระบุบทลงโทษหากทำไม่ได้ตามที่ตกลงไว้ จุดประสงค์คือทำให้ความคาดหวังชัดเจนและตรวจสอบได้จริง แต่ก็ต้องอ่านรายละเอียดให้ดี เพราะตัวเลข SLA ที่ดูดีไม่ได้การันตีว่าประสบการณ์ใช้งานจะดีเสมอไป หากมีเงื่อนไขหรือข้อยกเว้นซ่อนอยู่
